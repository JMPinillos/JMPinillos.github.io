                Introducción

                Las procesadores vectoriales proporcionan instrucciones que trabajan sobre vectores.

                Las operaciones sobre vectores presentan propiedades muy adecuadas para el procesamiento paralelo.

                Una sola instrucción de vector es equivalente a muchas posiciones escalares a la vez, ahorrando:

                • Control de bucle.
                • Posiciones de memoria para el indice.
                • Saltos tras cada iteración al comienzo del bucle.
                • Búsqueda de instrucciones.

                A cambio de esto necesitamos unidades funcionales y pipelines específicos para las operaciones vectoriales.

                 

                Arquitectura vectorial

                • Memoria principal común.

                • Unidad de carga y almacenamiento.

                • Unidades de ejecución especificas para realizar las distintas operaciones:

                  • Unidades vectoriales:

                    • Registros vectoriales.
                    • Unidades funcionales vectoriales.

                  • Unidad escalares:

                    • Registro escalares.
                    • Unidades funcionales escalares.

                   

                Ventajas:

                • No hay riesgos RAW.
                • Acceso a memoria rápido porque explota el principio de localidad.
                • Los riesgos de control disminuyen al no existir saltos.
                • Gran aprovechamiento del paralelismo.
                • Reducción del ancho de banda.

                 

                Sistemas de memoria

                En un procesador vectorial, los accesos a memoria son a colecciones de datos.

                Se utilizan 2 opciones de acceso a la memoria:

                • Memoria entrelazada: Diseño más sencillo y barato pero menos flexible. Un bus compartido.

                   

                • Bancos de memoria independientes: Es más cara pero más flexible. Buses dedicados.

                   

                • Memoria multibanco: Mezcla los dos sistemas anteriores.

                 

                Problemas en los procesadores vectoriales

                Hay tres problemas principales con los que podemos encontrarnos:

                • Problema 1: La longitud del vector en la aplicación no tiene por qué coincidir con el tamaño de los registros.

                  Solución: se añade un registro escalar llamado VLR (registro de longitud)

                  Este registro contiene el número de elementos que se utilizan en cada operación. Si un cálculo necesita vectores mayores que lo que cabe en el registro, se divide en bloques y se hace un bucle.

                   

                • Problema 2: El espaciado de un vector en memoria no tiene por qué ser 1 (es decir, los elementos de un vector pueden no estar contiguos en memoria).

                  Solución: Se indica el espaciado con un registro especifico de datos.

                  Si el espaciado es 1 el rendimiento es optimo, pero si es mayor el rendimiento disminuye.

                  Ejemplo: Si tenemos un espaciado de 8 y disponemos de 8 bancos, todos los accesos irán al mismo banco. Esto se resuelve haciendo que el espaciado y el numero de bancos sean primos entre sí.

                  Ademas, si ocurre una coincidencia, el compilador puede intervenir y añadir una columna hueca en la matriz.

                   

                • Problema 3: El bucle que se quiere vectorizar lleva dentro sentencias condicionales.

                  Solución: utilizar vectores de máscara.

                  Un vector máscara es un vector de valores booleanos. Si la máscara es 0, la operación no se ejecuta.

                   

                • Problema 4: DEPENDENCIAS.

                  Solución: ENCADENAMIENTO.

                  El encadenamiento es una técnica que consiste en solapar la ejecución de distintas instrucciones vectoriales aunque haya dependencias entre ellas.

                  Ejemplo: tenemos el siguiente par de instrucciones:

                  xxxxxxxxxx
                  2
                  1
                  multv v1,v2,v3  ;multiplica v2*v3 y el resultado lo guarda en v1.
                  2
                  addv v4,v1,v5   ;suma v1+v5 y el resultado lo guarda en v4.

                  Como hay dependencia no puede ejecutarse en paralelo.

                  Con el encadenamiento, a medida que se van generando componentes de V1, la suma puede irlas procesando. Seria cómo realizar una segmentación del vector y escalarla.

                 

                Vectorización del código

                Para poder vectorizar el código se tiene que cumplir dos condiciones:

                1. No debe haber dependencias de datos.
                2. El programa ha de estar expresado de forma que permita la vectorización al compilador.
