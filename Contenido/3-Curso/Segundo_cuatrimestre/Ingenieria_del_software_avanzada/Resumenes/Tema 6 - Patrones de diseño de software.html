Patrones de diseño de software

Introducción

Un patrón de diseño es una solución a un problema que se ha presentado en múltiples ocasiones y nos sirve de base para buscar soluciones a problemas de desarrollo del software.

Un patrón contiene una solución probada a un problema en un determinado contexto.

Un patrón de diseño:

  • Debe ser repetible: efectivo en más de una ocasión.
  • Descripción genérica: aplicable a múltiples contextos.

Los patrones pueden complicar el diseño, introduciendo clases adicionales, por ello es considerable utilizarlos en su justa medida.

Los patrones tienen como objetivo, aumentar la cohesión y disminuir el acoplamiento.

La información de los patrones se recopila en un catálogo de patrones.

Patrones de creación

Ayudan en la tarea de construcción de nuevos objetos de características complejas.

Fábrica abstracta (Abstract Factory)

Es un patrón de diseño creacional que nos permite producir familias de objetos similares sin especificar sus clases concretas.

Permite crear una interface de manipulación común para una familia de objetos.

Ejemplos:

  1. Videojuego con varios tipos de personajes que tienen operaciones comunes (saltar, atacar, correr...).
  2. Objetos de tipo mueble (Silla, Sofa, Mesa) y estos objetos a su vez están disponibles en variantes (Moderna, Victoriana, ArtDeco).

 

Constructor (Builder)

Es un patrón de diseño creacional que nos permite el mismo proceso de construcción para diferentes resultados.

Encapsula el método de construcción y devuelve una instancia concreta con las características seleccionadas.

Ejemplo:

  • Aplicación de conversión de vídeo:

    • Diferentes formatos de contendor( AVI, MPG, MOV...).
    • Diferentes códecs (audio/vídeo).
    • Operaciones adicionales (escalado, recortado...).

 

Prototipo (Prototype)

Es un patrón de diseño creacional que nos permite copiar objetos existentes sin que el código dependa de sus clases.

El prototipo implementa una interfaz clonar( ), creando copias idénticas o con ciertas modificaciones.

Ejemplos:

  1. Videojuego en el que creamos copias de un personaje.
  2. Editor de gráficos en el que es necesario crear nuevas formas con características comunes.
  3. El copy + paste de un programa.

 

Singleton

Es un patrón de diseño creacional que nos permite asegurarnos de que una clase tenga una única instancia, a la vez que proporciona un punto de acceso global a dicha instancia.

Ejemplos:

  1. Única instancia de una clase controladora de la ejecución de un programa.
  2. Clase para acceder a la base de datos y obtener información.
  3. Clase para obtener información de una API.

 

Patrones de estructura

Ayudan a ensamblar objetos en estructuras más grandes manteniendo la flexibilidad y eficiencia.

Adaptador (Adapter)

Es un patrón de diseño estructural que permite la colaboración entre objetos con interfaces incompatibles.

Se intercala una nueva clase que hace el trabajo de traducción o adaptación.

Ejemplos:

  1. Editor de dibujo que consume una librería de terceros para la creación de objetos gráficos.
  2. Una aplicación que consume información de diferentes APIs con mismos servicios pero diferente nomenclatura.

Hay 2 maneras de implementación:

OBJETO ADAPTADOR (composición)

 

CLASE ADAPTADORA (herencia)

 

Compuesto (Composite)

Es un patrón de diseño estructural que permite componer objetos en estructuras de árbol y trabajar con esas estructuras como si fueran objetos individuales.

Necesitamos una interface de manipulación común.

Se define una clase abstracta que representa tanto a las primitivas como sus contenedores.

Ejemplos:

  1. Jerarquía de comentarios en un blog.
  2. Editor de dibujo que permite agrupar formas jerárquicamente.

 

Fachada (Facade)

Es un patrón de diseño estructural que proporciona una interface unificada para interactuar con un conjunto de interfaces de otros objetos.

Es similar al adaptador, pero el adaptador adapta una clase concreta, la fachada esta pensada para reducir el acoplamiento.

Ejemplos:

  1. Cualquier aplicación que trabaje con un amplio grupo de objetos que pertenecen a una sofisticada biblioteca o framework.

 

Proxy

Es un patrón de diseño estructural que permite proporcionar un sustituto o marcador de posición para otro objeto. Un proxy controla el acceso al objeto original, permitiendo hacer algo antes o después de que la solicitud llegue al objeto original.

El proxy (apoderado) sustituye al objeto real (sujeto) y permite manipularlo de manera indirecta.

Ejemplos:

  1. Operaciones de acceso a red, en las que el proxy actúa como fachada o adaptador.
  2. Sustitución temporal de objetos cuya carga en memoria resulta costosa, retrasando su carga hasta que sea imprescindible (lazy load).

 

 

Patrones de comportamiento

Utiles para trabajar con algoritmos y repartir responsabilidades entre diferentes objetos.

Cadena de responsabilidad (Chain of Responsibility)

Es un patrón de diseño de comportamiento que te permite pasar solicitudes a lo largo de una cadena de manejadores. Al recibir una solicitud, cada manejador decide si la procesa o si la pasa al siguiente manejador de la cadena.

Ejemplos:

  1. Ayuda contextual en una aplicación:

    • Buscar información más específica según el elemento consultado.

  2. Proceso de eventos del ratón en un navegador.

 

Mediador (Mediator)

Es un patrón de diseño de comportamiento que te permite reducir el acoplamiento. El patrón restringe las comunicaciones directas entre los objetos, forzándolos a colaborar únicamente a través de un objeto mediador que controla y coordina las comunicaciones.

Ejemplos:

  1. Control y coordinación de varios elementos de interfaz gráfica.
  2. Un simulador de una red logística, que conoce el estado global del sistema.

 

Observador (Observer)

Es un patrón de diseño de comportamiento que te permite notificar a varios objetos sobre cambios que se producen en otro objeto.

Es un problema parecido al mediador, pero ahora solo se trata de notificar.

El patrón define dos tipos de objetos:

  • Sujeto: donde se producen los cambios.
  • Observador: es notificado y propaga la información.

Ejemplos:

  1. Un sistema de control domótico.

 

Estado (State)

Es un patrón de diseño de comportamiento que permite a un objeto alterar su comportamiento cuando su estado interno cambia.

El patrón define una clase asociada que defina la funcionalidad concreta en cada caso.

Ejemplos:

  1. Un videojuego en el que el comportamiento de un personaje depende de variables internas.
  2. El control de un proceso, donde se utilizan diferentes mecanismos de control en función del estado (Normal, Emergencia, Sobrecarga...).

 

Método plantilla (Template-method)

Es un patrón de diseño de comportamiento que define el esqueleto de un algoritmo en la superclase pero permite que las subclases sobrescriban pasos del algoritmo sin cambiar su estructura.

El patrón define una clase base con un método que implementa el esqueleto del algoritmo (método plantilla).

  • Emplea métodos abstractos que deben definirse en cada clase derivada.
  • Se pueden agregar funcionalidades específicas.
  • La secuencia general de pasos lógicos queda definida.

Ejemplos:

  1. Visor de documentos que soporta varios tipos de archivo.
